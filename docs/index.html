CLIP VLM 101

How a vision-language model turns images + prompt templates into embeddings, fuses them into one vector, stores them in a database, and answers searches — with the whole pipeline running live in your browser below. Read the code on GitHub →

1 · How it works

Both CLIP encoders project into the same vector space, so a cat photo and the sentence “a photo of a cat” land close together. Everything below is dot products between unit vectors.

Imageuploaded, or one of the 14 examples
→
Vision encoderimage_emb — 512 numbers: what it looks like
→
Prompt template“a photo of a {tag}” × vocabulary → top tags = meta tags
→
Text encodertext_emb — 512 numbers: what its caption means
→
Concatenatefused_emb = [image ; text] / √2 — 1024 numbers, both signals
→
DatabaseSQLite row: path · caption · tags · 3 vectors
→
Retrievalquery → embedding → dot products → top-k

2 · The database

These 14 images were ingested by ingest.py: embedded, auto-tagged with the prompt template (zero-shot — no training), captioned, and stored. Every SQLite row carries exactly three vectors (this page loads the exported image_emb + text_emb and recomputes the fused score as their average — same math):

vectorshapedtypebytes as BLOBmeaning
image_emb(512,)float32, unit-length2 048what the image looks like (vision tower)
text_emb(512,)float32, unit-length2 048what its caption/tags mean (text tower)
fused_emb(1024,)float32, unit-length4 096[image_emb ; text_emb] / √2 — both signals

5 · Read the code

One concept per file. The browser demo mirrors the Python pipeline with matching module names.

Python pipelinewhat it teaches
templates.pyprompt templates: sentences with holes, filled per candidate tag
embedder.pyCLIP → unit-length 512-d vectors for images and text
tagger.pyzero-shot meta tags = dot products + argsort, no training
fusion.pythe concatenation: [image ; text] / √2, and why the math works
features.pythe one-call API: image → meta tags + all three embeddings
db.pyvectors as float32 BLOBs in plain SQLite
ingest.pycomposes the pipeline: embed → tag → caption → fuse → store
search.pytext / image / fused retrieval with dot products
export_web.pydump the SQLite gallery to the db.json this page uses
…and its browser twinsame pipeline, in JavaScript
js/templates.jsmirror of templates.py — same template, same vocabulary
js/clip.jsmirror of embedder.py — the same model via transformers.js
js/rank.jsmirror of tagger.py + fusion.py + search.py — the pure math
js/app.jswires the modules to this page (gallery, search, upload)
