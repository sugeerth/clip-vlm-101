CLIP VLM 101

How a vision-language model turns images + prompt templates into embeddings, fuses them into one vector, stores them in a database, and answers searches — with the whole pipeline running live in your browser below. Read the code on GitHub →

1 · How it works

Both CLIP encoders project into the same vector space, so a cat photo and the sentence “a photo of a cat” land close together. Everything below is dot products between unit vectors.

Imageuploaded, or one of the 14 examples
→
Vision encoderimage_emb — 512 numbers: what it looks like
→
Prompt template“a photo of a {tag}” × vocabulary → top tags = meta tags
→
Text encodertext_emb — 512 numbers: what its caption means
→
Concatenatefused_emb = [image ; text] / √2 — 1024 numbers, both signals
→
DatabaseSQLite row: path · caption · tags · 3 vectors
→
Retrievalquery → embedding → dot products → top-k

2 · The database

These 14 images were ingested by ingest.py: embedded, auto-tagged with the prompt template (zero-shot — no training), captioned, and stored. This page loads that exported database.
