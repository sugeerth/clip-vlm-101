What you're seeing

This is structured orchestration: a typed graph of agents where every hand-off carries a contract (the output keys it must produce). The runtime topologically orders the nodes, runs each once its inputs are ready, and validates every output against its contract. A node that breaks its contract — or any node downstream of one — is skipped: the graph fails closed instead of passing bad data along.

The panel is a fan-out: it spawns one sub-agent per rubric (relevance, specificity, faithfulness), each returning a structured vote. Pick the rogue sub-agent case and watch it emit the wrong shape and get dropped — the council still rules from the surviving votes. When the panel is too split to rule, the graph escalates to a bounded-confidence debate; if the judges still split into factions, the honest answer is to abstain, dissenters named. Nothing here is faked — it's flow.js executing live, the same code the Python twin runs.
